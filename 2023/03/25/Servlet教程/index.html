servlet

概念

  • 运行在服务器上的一个小程序
  • servlet ——-servlet applet
  • servlet就是一个接口，定义了Java类被服务器识别的规则。
  • 将来我们自定义一个类，实现servlet接口，复写方法。

快速入门

  • 创建Java EE 项目
  • 定义一个类，实现Servlet接口
  • 实现接口中的方法
  • 配置Servlet

Servlet的执行原理

  • 服务器接受客服端浏览器请求，会解析url路径，获取访问的Servlet的路径资源
  • 查找web.xml文件，是否有对应的标签内容
  • 如果有，就会找到对应的全类名
  • tomcat会将字节码文件加载进内存，并创建对象，调用方法

Servlet的生命周期

  • 被创建

    • init方法， 只在servlet被创建的时候执行，只执行一次

      • Servlet 什么时候被创建———-取决于标签下数字 数字的大小 ，小于0的时候时Servlet被访问的时候被创建，大于等于0时候时服务器启动的时候被创建。不配置这个选项，默认时Servlet被访问时，才会被创建

      • Servlet的init方法只会执行一次，说明Servlet是单例的

  • 提供服务

    • service 方法，每访问一次Servlet就执行一次

  • 被销毁

    • destory 方法，Servlet被销毁时执行。在服务器正常关闭时，Servlet被销毁，执行一次。
    • 一般用于释放资源

Servlet3.0

  • 优点

    • 支持注解配置。可以不需要web.xml配置了

  • 步骤

    • 创建javaEE 项目，选择Servlet3.0以上版本

    • 定义类，实现Servlet接口，复写方法

    • 在类上使用@WebServlet注解，进行配置

      • @WebServlet(urlPatterns ="/d")
@WebServlet(value = "/d")
@WebServlet("/d")

Web相关概念

    1. 软件架构
      1. C / S : 客服端 / 服务器端
      1. B / S :浏览器 / 服务器端
    1. 资源分类
      • 静态资源：所有用户访问后，得到的结果都是一样的，称为静态资源，静态资源可以直接被浏览器解析
        • 如html,css,javascript
      • 动态资源：每个用户访问相同的资源后，得到的结果可能不一样，称为动态资源，动态资源被访问后，需要先转换为静态资源，在返回给浏览器。
        • 如 servlet/jsp,php,asp

  • 3 . 网络通信三要素

  • 1，ip : 电子设备（计算机）在网络中的唯一标识

  • 2，端口：应用程序在计算机中的唯一标识 0~65536，尽量不要使用1024之前的端口，可能已经被系统所占用。

  • 传输协议 ： 规定了数据传输的规则

    • 基础协议
      • tcp : 安全协议， 速度慢
      • udp:不安全协议 ， 速度快

Servlet_urlpartten配置

  • xml 文件配置
  • @WebServlet 配置
    • 一个Servlet可以定义多个访问路径： ```@WebServlet({“/d”,”/dd4”,”/dddd4”})
    • 定义路径的规则
      • /xxx
      • /xxx/xxxx
      • *.do

HTTP :

  • 概念：Hyper Text Transfer Protocol 超文本传输协议
    • 传输协议：定义了，客户端和服务器通信时，发送数据的格式
    • 特点
        1. 基于TCP/IP的高级协议
        2. 默认端口号：80
        3. 基于请求/响应模式： 一次请求对应一次响应
        4. 每次请求之间相互独立，不能交互数据
